# CYB333_Project_Environment_WV
Windows Security Baseline Compliance Automation Script
Project Overview
This project automates the assessment of a Windows system against a defined security baseline. The script evaluates key security controls such as firewall status, password policies, Windows Update configuration, installed patches, and other system hardening settings. Results are logged with timestamps and pass/fail indicators, providing a repeatable and efficient method for identifying misconfigurations.
This project was developed as part of CYB 333 – Security Automation, demonstrating how automation can streamline security operations, reduce manual workload, and support continuous monitoring.

Objectives
•	Automate the evaluation of Windows security baseline controls
•	Provide clear pass/fail results for each check
•	Log findings to a timestamped output file
•	Improve consistency and repeatability in security assessments
•	Demonstrate practical application of security automation concepts

Key Features
•	Modular Python functions for each security check
•	Logging of results with timestamps
•	Error handling for missing permissions or inaccessible registry keys
•	Console output for real time visibility
•	Expandable structure for adding future checks
•	Designed to run on Windows environments

How It Works
The script performs a series of automated checks, including:
•	Windows Firewall status
•	Password policy configuration
•	Windows Update settings
•	Installed updates
•	Running services
•	Local administrator accounts
•	Audit policy settings
Each check returns:
•	PASS – compliant with baseline
•	FAIL – deviation detected
•	ERROR – insufficient permissions or inaccessible data
All results are logged for documentation and review.
